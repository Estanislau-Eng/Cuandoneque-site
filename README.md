# 🌐 Website Institucional — Cuandoneque

> Website institucional desenvolvido com foco em organização de código, arquitetura MVC e desenvolvimento web responsivo.

## Sobre o projeto

O **Cuandoneque-site** é um projeto de desenvolvimento web criado para a construção de um website institucional moderno, organizado e responsivo.

O projeto foi desenvolvido com uma preocupação especial na **estruturação do código e separação de responsabilidades**, utilizando uma abordagem baseada no padrão **Model-View-Controller (MVC)**.

Mais do que criar uma interface visual, este projeto teve como objetivo aplicar boas práticas de **organização, reutilização e manutenção de código**.

---

## Objetivos

* Desenvolver um website institucional responsivo;
* Organizar o projeto de forma modular;
* Aplicar o padrão arquitetural MVC;
* Separar dados, lógica e apresentação;
* Criar componentes reutilizáveis;
* Aplicar boas práticas de desenvolvimento web;
* Melhorar a organização e a manutenção do código.

## Arquitetura do projeto

O projeto utiliza uma organização baseada no padrão **MVC — Model-View-Controller**.

```text
Cuandoneque-site/
│
├── imagens/
├── paginas/
│
├── src/
│   ├── models/
│   ├── controllers/
│   ├── views/
│   │   ├── pages/
│   │   └── components/
│   │
│   ├── assets/
│   │   ├── css/
│   │   └── js/
│   │
│   └── utils/
│
├── .gitignore
├── config.json
├── index.html
└── README.md
```

### 🔹 Models

Responsáveis pela organização dos dados e informações utilizadas na aplicação.

### 🔹 Views

Responsáveis pela apresentação e estrutura das páginas e componentes HTML.

### 🔹 Controllers

Responsáveis pela lógica de interação e comunicação entre os diferentes componentes da aplicação.

### 🔹 Assets

Contém os recursos estáticos, como ficheiros CSS e JavaScript.

### 🔹 Utils

Reúne funções auxiliares e validadores reutilizáveis.

## Como executar

1. Clone o repositório:

```bash
git clone https://github.com/Estanislau-Eng/Cuandoneque-site.git
```

2. Aceda à pasta do projeto:

```bash
cd Cuandoneque-site
```

3. Abra o ficheiro `index.html` no navegador.

> Para uma melhor experiência de desenvolvimento, recomenda-se utilizar um servidor local.
